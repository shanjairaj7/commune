How it works

One loop, one schema, one place to deliver and respond safely.

Receive

Webhook intake for Slack and email. Verify signatures, parse data, and store raw payloads.

Normalize

Every message becomes a single shape with thread_id, participants, and content.

Respond

Reply in-thread with attachments and clear attribution, across channels.

Unified message model

No channel-specific logic. Your agent always receives the same payload.

Agent-ready payload

Stable fields for routing, permissions, and thread context. Built for async work.

  • thread-first storage with reliable reply semantics
  • consistent identity across channels
  • attachments tracked separately for safe retrieval
